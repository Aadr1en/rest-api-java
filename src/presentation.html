Présentation - API REST Java Sans Framework

Contexte du Projet

Dans une startup, le projet vise à développer un service d'API REST simple en Java pour gérer une file d'attente de billets, comme dans les banques (BOA, BMOI) ou les espaces clients (Yas, Orange). L'objectif est d'assurer un contrôle total et une légèreté maximale en utilisant Java pur, sans frameworks comme Spring Boot ou Jakarta EE.

Exemple de billet traité :

Agence Yas Ankorondrano
Ticket N°: 001
Accueil/Guichet N°: 05
Il y a encore 8 personnes en attente.

Objectifs à Atteindre

  • Développer une API REST en Java sans frameworks.
  • Dockeriser une application Java.
  • Versionner et publier le projet sur GitHub.
  • Déployer gratuitement une API conteneurisée.

Contraintes Techniques

  • Langage : Java (version 17 minimum recommandée).
  • Sans framework : utiliser uniquement le SDK Java standard (ex. com.sun.net.httpserver.HttpServer).
  • Build : utiliser javac ou un outil comme Gradle ou Maven.
  • Stockage : en mémoire (ArrayList, HashMap...).
  • Persistance : fichier texte ou SQLite (optionnel).
  • Script de test automatique.
  • Pipeline CI/CD avec GitHub Actions.
  • Frontend HTML minimal (optionnel).

Étapes de Développement

  • Développement de l'API : Implémentation d'une classe FIFO avec les méthodes enqueue, dequeue, peek, isEmpty, size.
  • Dockerisation : Création d'un Dockerfile pour la compilation et l'exécution.
  • Git & GitHub : Initialisation du dépôt, ajout de .gitignore, push sur GitHub.
  • Déploiement : Déploiement sur une plateforme gratuite (Render, Railway).

Technologies Utilisées

  • Java 17
  • Docker
  • GitHub Actions
  • Git

Conclusion

Ce projet démontre la création d'une API REST légère et fonctionnelle pour la gestion de files d'attente, prête à être déployée et utilisée dans un environnement de production.

REST API Java – File d'attente

Ce projet expose une API REST minimaliste permettant de gérer des tickets dans une file FIFO (First In, First Out), développée entièrement en Java pur, sans framework.

Technologies utilisées

  • Java 17+
  • Docker (déploiement sur Render)
  • Git & GitHub (versionnement)
  • curl ou Postman (tests d'API)

Endpoints disponibles

  • POST /enqueue : Génère automatiquement un ticket
  • GET /peek : Affiche le ticket en tête de file
  • GET /dequeue : Retire le ticket suivant
  • GET /size : Retourne la taille actuelle de la file
  • GET /isempty : Vérifie si la file est vide

Déploiement Docker

Ce projet est entièrement conteneurisé avec Docker et déployé sur Render.

Port exposé : 8080
